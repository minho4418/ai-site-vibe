# 데일리 브리핑 루틴 — 소셜(캐주얼) 필드 생성 지침

클라우드 "데일리 브리핑" 루틴이 출력하는 브리핑 JSON에 **선택 필드 두 개**(`socialHook`,
항목별 `social`)를 추가로 생성한다. 이 필드는 **Threads 자동 게시(`src/lib/threads.ts`)에서만**
쓰이고, **오늘의 브리핑 페이지·JSON-LD 렌더에는 전혀 영향이 없다**(`BriefingView`/`briefingJsonLd`는
읽지 않음). 없으면 본문에서 자동 폴백되므로 하위호환된다.

아래 블록을 루틴 프롬프트의 "JSON 출력 스펙" 부분에 그대로 추가한다.

---

## 루틴 프롬프트에 추가할 지침 (복붙용)

브리핑 JSON에 아래 **선택 필드**를 함께 채워라. Threads 자동 게시에만 쓰이고 사이트 렌더에는
영향이 없다.

1. 최상위 `socialHook` (문자열, 선택) — **가장 중요. 첫 글이 전체 도달을 좌우한다.**
   - 스레드 맨 위에 단독으로 올라가는 "훅".
   - **제목(title)을 그대로 반복하지 말 것.** 그날 가장 충격적인 한 가지(놀라운 수치·이름·반전)를
     앞세워 스크롤을 멈추게 만들어라.
   - 임팩트 공식: **[구체적 수치/놀라운 사실] + [그래서 뭐가 좋은지 또는 궁금증]**
   - 캐주얼·구어체. 이모지 0~1개. **URL·링크 금지**(맨 윗글에 링크가 있으면 도달이 깎인다).
   - **펀치 한 줄만 써라(1~2줄, 100자 이내).** 날짜·"오늘의 AI 뉴스 한눈에 🧵 #AI…" 태그라인은
     코드가 **자동으로 아래 줄에 붙이니** socialHook 에 날짜·🧵 마무리를 넣지 마라.
   - 좋은 예(펀치만):
     - "100만 토큰을 통째로 읽는 프런티어 모델이 '오픈웨이트'로 공짜로 풀렸어요 🤯"
     - "애플이 자사 AI 모델을 개발자에게 '무료'로 열었습니다"
   - 나쁜 예(밋밋함·제목 반복): "오늘 AI 뉴스 정리 / 오픈웨이트 모델 등장, 애플 개발 문턱 낮춰"

2. 각 항목(item)에 `social` (문자열, 선택)
   - 그 뉴스 한 건을 **친구에게 말하듯** 캐주얼하게 풀어쓴 1~2문장.
   - 전문용어는 풀어서 쉽게. 정확성은 유지(과장 금지).
   - **번호/불릿 붙이지 말 것**(자동으로 1️⃣·2️⃣… 가 앞에 붙는다).
   - **출처 넣지 말 것**(자동으로 `— 출처`가 뒤에 붙는다). **URL 도 넣지 말 것**(본문 글엔 외부 링크 없음).
   - **60~110자 권장**, 한 항목 1~2문장(120자에서 자동으로 더 줄어듦). 길면 스크롤 이탈.
   - Threads에는 전체 항목 중 **앞에서 5개까지만** 실리므로, 최소한 앞쪽 5개에는 채워라.

3. 저작권 안전 규칙 (필수 — `social`·`socialHook`·`text` 모두에 적용)
   - **원문 문장을 그대로 옮기지 말 것.** 사실(무엇이·언제·수치·이름)만 추출해 완전히 새 문장으로 재작성하라.
   - **기사 전문 재현 금지.** 한 항목은 2~3문장 요약으로 제한한다.
   - **분석·칼럼·피처 기사의 독창적 표현·비유·평가**는 인용하지 말고, 사실만 전하라.
   - 출처 URL 은 항상 정확히 채워 원본으로 링크가 가게 하라(귀속 + 트래픽 환원).

없으면: `socialHook` 미입력 시 그날 헤드라인(title)을 앞세운 기본 훅으로,
`social` 미입력 시 본문 `text` 전체로 자동 대체된다.

## 출력 JSON 예시 (social/socialHook 포함)

```json
{
  "date": "2026-06-19",
  "title": "오픈웨이트 프런티어 모델 MiniMax M3 등장, 애플은 AI 개발 문턱 낮춰",
  "summary": "오픈웨이트 코딩 모델 MiniMax M3가 100만 토큰 컨텍스트로 등장했고…",
  "socialHook": "100만 토큰을 통째로 읽는 프런티어 모델이 '오픈웨이트'로 공짜로 풀렸어요 🤯",
  "sections": [
    {
      "heading": "주목할 AI 툴/제품",
      "items": [
        {
          "text": "MiniMax M3가 100만 토큰 컨텍스트와 코딩·멀티모달을 한 모델에 담은 첫 오픈웨이트 프런티어 모델로 공개됐다. 입력 100만 토큰당 0.60달러로 저렴하다.",
          "url": "https://www.minimax.io/blog/minimax-m3",
          "source": "MiniMax",
          "social": "MiniMax M3가 오픈웨이트로 풀렸어요. 100만 토큰을 통째로 읽어서 긴 코드베이스 분석이나 자동화 에이전트에 그대로 물리기 딱 좋고, 가격도 100만 토큰당 0.6달러로 쌉니다."
        }
      ]
    }
  ],
  "pick": { "text": "…", "url": "…" }
}
```

## 게시 결과 매핑(참고)

- `socialHook`(펀치 한 줄) → 1번째 글 = `socialHook` + 자동 `🗓 M/D 오늘의 AI 뉴스 한눈에 🧵 #AI #생성형AI #개발자`
  (없으면 첫 줄은 `title`로 폴백, 날짜 줄은 동일하게 자동 추가) · **링크 없음**
- 항목 `social`(없으면 `text`) → `1️⃣ {짧은 요약}` + 줄바꿈 + `— {source}` (앞 5개, **외부 링크 없음**)
- 마지막 글 → 자사 사이트 링크 1개(`{사이트}/daily/{date}`) + 팔로우 CTA — **체인에서 유일한 외부 링크**

## 노출·유입 전략 (설계 의도)

Threads 는 **외부 링크가 있는 글의 도달을 크게 깎는다.** 또 각 항목에 원문 링크를 달면 클릭이
**출처(타사)로 새어** 자사 사이트(knewittoday.com) 유입에 도움이 안 된다. 그래서:

1. **훅·본문 글엔 링크를 넣지 않는다** → 도달(노출) 보존. 본문 자체로 읽을 가치를 준다(저장·공유 유도).
2. **외부 링크는 마지막 글의 자사 사이트 1개로 집중** → 원문이 궁금한 사람도 우리 사이트를 거치게 →
   유입을 우리 쪽으로 모은다(거기서 원문 링크 제공).
3. **첫 줄(훅)이 전부**: 구체적 수치·반전으로 스크롤을 멈추게. 제목 반복 금지.
4. **팔로우 CTA + 매일 같은 시각** → 재방문·알고리즘 가산. 마지막 글에서 팔로우를 유도한다.
5. (선택) 훅이나 마지막 글에 **토픽 해시태그 1개**(#AI 등)로 탐색 노출 보강 — 과용 금지.

## 날짜(`date`) 주의 — 매일 발행 핵심

`date` 하나가 아래 전부를 좌우하므로 **반드시 한국시간(Asia/Seoul) 기준 오늘 날짜**를 `YYYY-MM-DD`
로 넣어야 한다. UTC 로 계산하면 한밤~새벽 발행 시 **하루 어긋날 수 있다**(8시 KST = 전날 23시 UTC).

- 훅의 `M/D` 표기 (`date` 에서 파생)
- 본문 페이지/마지막 글 링크 `…/daily/{date}`
- JSON-LD 발행일(`{date}T08:00:00+09:00`)
- **멱등 키**: `daily_briefings.threads_post_id` 는 날짜당 1행 → 같은 `date` 면 재게시 안 함.
  날짜가 매일 새로 바뀌어야 매일 1회 정상 게시된다(루틴 프롬프트에 "오늘(KST) 날짜" 명시 필수).
